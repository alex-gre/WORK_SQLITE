SQLite Краткое руководство

❮ Предыдущая Следующая Глава ❯

SQLite является библиотекой, которая реализует автономный, бессерверную, нулевой конфигурации, транзакционной SQL движок базы данных. SQLite является одним из наиболее быстро растущих СУБД вокруг, но это рост с точки зрения популярности, а не что-то делать с его размером. Исходный код SQLite находится в свободном доступе.

Что такое SQLite?

SQLite является библиотекой в ​​процессе, который реализует автономный, Serverless, нулевой конфигурации, транзакционной SQL движок базы данных. Это одна база данных, которая равна нулю сконфигурированные, это означает, что, как и другие базы данных, вам не нужно настроить его в вашей системе.

SQLite двигатель не является отдельным процессом, как и другие базы данных вы можете связать его статически или динамически согласно вашему требованию с приложением. SQLite получает доступ к своим файлам для хранения непосредственно.

Почему SQLite?

  • SQLite не требует отдельного процесса сервера или системы , чтобы operate(erverless) .

  • SQLite поставляется с нулевой конфигурацией, которая означает отсутствие настройки и администрирования необходимо.

  • Полная база данных SQLite хранятся в одном файле на диске кросс-платформенного.

  • SQLite очень маленький и легкий вес, менее чем 400KiB полностью сконфигурирован или менее 250KiB с дополнительными функциями опущены.

  • SQLite является автономным, который означает, что нет внешних зависимостей.

  • SQLite сделка полностью ACID-совместимая, позволяя безопасный доступ из нескольких процессов или потоков.

  • SQLite поддерживает большинство возможностей языка запросов , найденных в SQL92 (SQL2) стандарта.

  • SQLite написана в ANSI-C и обеспечивает простой и легкий в использовании API.

  • SQLite доступен на UNIX (Linux, Mac OS-X, Android, iOS) и ОС Windows (Win32, WinCE, WinRT) .

История:

  1. 2000 - D. Ричард Хипп разработал SQLite с целью не введения необходимого для работы программы.

  2. 2000 - В августе SQLite 1.0 Выпущено с GNU Database Manager.

  3. 2011 - Hipp объявил добавить интерфейс UNQl к БД SQLite и разработать UNQLite (Document oriented database) .

SQLite - Установка

SQLite славится своей большой особенностью нулевой конфигурации, что означает отсутствие сложной конфигурации или требуется введение. Эта глава проведет вас через процесс настройки SQLite на Windows, Linux и Mac OS X.

Установка SQLite В Windows

  • Перейти к SQLite загрузки страницы и загружать скомпилированные двоичные файлы из раздела Windows.

  • Вам нужно будет загрузить sqlite-shell-win32-*.zip и sqlite-dll-win32-*.zip - sqlite-dll-win32-*.zip заархивированные файлы.

  • Создайте папку C: \> SQLite и разархивировать выше двух сжатых файлов в этой папке, которая даст вам sqlite3.def, sqlite3.dll и sqlite3.exe файлы.

  • Добавить C: \> SQLite в вашей переменной PATH среды и , наконец , перейти в командную строку и выдает sqlite3 команду , которая должна отображать результат что - то , как показано ниже. 

C:\>sqlite3
SQLite version 3.7.15.2 2013-01-09 11:53:05
Enter ".help" for instructions
Enter SQL statements terminated with a ";"
sqlite>

Установка SQLite В Linux

Сегодня почти все ароматы операционной системы Linux в настоящее время поставляются с SQLite. Таким образом, вы просто выполнить следующую команду, чтобы проверить, если у вас уже есть SQLite установлена ​​на вашем компьютере или нет. 

$sqlite3
SQLite version 3.7.15.2 2013-01-09 11:53:05
Enter ".help" for instructions
Enter SQL statements terminated with a ";"
sqlite>

Если вы не видите выше результата, то это означает, что вы не SQLite, установленные на вашем компьютере Linux. Итак, давайте следовать следующим шагам для установки SQLite:

  • Перейти на SQLite страницу загрузки и скачать sqlite-autoconf-*.tar.gz из исходного кода раздела.

  • Выполните следующие действия: 

$tar xvfz sqlite-autoconf-3071502.tar.gz
$cd sqlite-autoconf-3071502
$./configure --prefix=/usr/local
$make
$make install

Выше процедура завершится с SQLite установки на машине Linux, которые вы можете проверить, как описано выше.

Хотя последняя версия Mac OS X поставляется с предварительно установленным с SQLite, но если у вас нет установки доступной затем выполните следующие действия.

SQLite - Синтаксис

SQLite сопровождается уникальным набором правил и руководств, называемых Синтаксис. Этот учебник дает вам быстрый старт с SQLite, перечислив все основные SQLite Синтаксис:

Чувствительность к регистру

Важный момент следует отметить, что SQLite является case insensitive к GLOB glob case insensitive , но их некоторая команда , которая чувствительна к регистру , как GLOB и glob имеет разный смысл в SQLite заявления.

Комментарии

SQLite комментарии дополнительные примечания, которые вы можете добавить в свой код SQLite, чтобы увеличить читаемость и они могут появляться в любом месте пробелы могут возникнуть, в том числе внутри выражений и в середине других операторов SQL, но они не могут быть вложенными.

SQL комментарии начинаются с двух подряд "-" символов (ASCII 0x2d) и продлить вплоть до следующего символа новой строки (ASCII 0x0a) или до окончания ввода, что наступит первым.

Вы можете также использовать комментарии C-стиля , которые начинаются с "/*" и расширить до и включая следующий "*/" символ пары или до окончания ввода, что наступит первым. комментарии C-стиля могут занимать несколько строк. 

sqlite>.help -- This is a single line comment

SQLite заявления

Все SQLite заявления начать с любым из ключевых слов , как SELECT, INSERT, UPDATE, DELETE, ALTER, DROP и т.д. , и все заявления , заканчиваются точкой с запятой (;) .

SQLite ПРОАНАЛИЗИРУЙТЕ заявление: 

ANALYZE;
or
ANALYZE database_name;
or
ANALYZE database_name.table_name;

SQLite И / ИЛИ Раздел: 

SELECT column1, column2....columnN
FROM   table_name
WHERE  CONDITION-1 {AND|OR} CONDITION-2;

SQLite ALTER TABLE заявление: 

ALTER TABLE table_name ADD COLUMN column_def...;

SQLite ALTER TABLE Заявление (Rename) : 

ALTER TABLE table_name RENAME TO new_table_name;

SQLite ATTACH DATABASE заявление: 

ATTACH DATABASE 'DatabaseName' As 'Alias-Name';

SQLite BEGIN TRANSACTION заявление: 

BEGIN;
or
BEGIN EXCLUSIVE TRANSACTION;

SQLite МЕЖДУ пункта: 

SELECT column1, column2....columnN
FROM   table_name
WHERE  column_name BETWEEN val-1 AND val-2;

SQLite COMMIT заявление: 

COMMIT;

SQLite CREATE INDEX заявление: 

CREATE INDEX index_name
ON table_name ( column_name COLLATE NOCASE ) ;

SQLite CREATE UNIQUE INDEX заявление: 

CREATE UNIQUE INDEX index_name
ON table_name ( column1, column2,...columnN) ;

SQLite CREATE TABLE заявление: 

CREATE TABLE table_name(
   column1 datatype,
   column2 datatype,
   column3 datatype,
   .....
   columnN datatype,
   PRIMARY KEY( one or more columns )
);

SQLite CREATE TRIGGER заявление: 

CREATE TRIGGER database_name.trigger_name

SQLite - Тип данных

SQLite тип данных атрибут, который определяет тип данных любого объекта. Каждый столбец, переменная и выражение, связанных с типом данных в SQLite.

Вы могли бы использовать эти типы данных при создании таблиц. SQLite использует более общую систему динамического типа. В SQLite, тип данных значения связан с самим значением, а не с его контейнера.

Классы SQLite для хранения:

Каждое значение, которое хранится в базе данных SQLite имеет один из следующих классов хранения:

Storage Class Описание
NULL Значение представляет собой значение NULL.
INTEGER Значение представляет собой целое число со знаком, хранится в 1, 2, 3, 4, 6 или 8 байт в зависимости от величины значения.
REAL Значение представляет собой значение с плавающей точкой, хранится в виде 8-байтовом IEEE числа с плавающей точкой.
TEXT Значение представляет собой текстовую строку, хранятся с использованием кодировки базы данных (UTF-8, UTF-16BE or UTF-16LE)
BLOB Значение представляет собой двоичный объект данных, хранится точно так, как это было ввод.

SQLite класс хранения немного более общим, чем тип данных. Класс хранения INTEGER, например, включает в себя 6 различных целочисленных типов данных различной длины.

SQLite - ATTACH базы данных

Рассмотрим случай, когда у вас есть несколько баз данных доступны, и вы хотите использовать какой-либо один из них в то время. SQLite ATTACH DTABASE оператор используется для выбора конкретной базы данных и после этой команды, все SQLite заявления будут выполнены в соответствии с прилагаемой базой данных.

Синтаксис:

Базовый синтаксис SQLite ATTACH DATABASE заявление выглядит следующим образом: 

ATTACH DATABASE 'DatabaseName' As 'Alias-Name';

Над командой будет также создать базу данных в случае, если база уже не создан, в противном случае он будет просто прикрепить имя файла базы данных с логической базы данных «Алиас-Name».

SQLite - база данных DETACH

SQLite DETACH DTABASE оператор используется для отсоединения и диссоциирует с именем базы данных из соединения с базой данных , которая была ранее присоединенным с помощью ATTACH заявления. Если же файл базы данных был присоединен с несколькими псевдонимами, то DETACH команда отключит только имя и остальную часть прикрепленного будет еще продолжаться. Вы не можете отделить main или temp базы данных.

Если база данных находится в оперативной памяти или временной базы данных, база данных будет разрушена, а ее содержимое будет потеряно.

Синтаксис:

Основной синтаксис заявления SQLite DETACH DATABASE «псевдоним Name» выглядит следующим образом: 

DETACH DATABASE 'Alias-Name';

Вот Алиас-Name 'тот же псевдоним, который вы использовали при присоединении базы данных с помощью ATTACH заявления.

SQLite - Выражения

Выражение представляет собой сочетание одного или нескольких значений, операторов и функций SQL, которые оценивают на значение.

SQL выражения, как формулы и они написаны на языке запросов. Вы можете также использовать для запроса к базе данных для конкретного набора данных.

Синтаксис:

Рассмотрим основной синтаксис ЗЕЬЕСТ следующим образом: 

SELECT column1, column2, columnN 
FROM table_name

Существуют различные типы SQLite выражения, которые упомянуты ниже:

SQLite - логические выражения:

SQLite Булевы выражения выборки данных на основе сопоставления одного значения. Ниже приводится синтаксис:

Здесь numerical_expression используются для математического выражения или любой формулы. Ниже приведен простые примеры, показывающие использование SQL числовых выражений:

Оператор AND:

AND оператор допускает существование нескольких условий WHERE оговорки о SQLite инструкции. При использовании и оператор, полное условие будет считать истинным, когда все условия истинны. Так, например [условие1] и [условие2] будет справедливо только тогда, когда обе условие1 и условие2 являются истинными.

Синтаксис:

Базовый синтаксис и оператор с ИНЕКЕМ выглядит следующим образом: 

SELECT column1, column2, columnN 
FROM table_name
WHERE [condition1] AND [condition2]...AND [conditionN];

Вы можете объединить N число условий с помощью и оператора. Для действия, которые необходимо принять в SQLite заявления, будь то сделка или запрос, все условия, которые отделены друг от И должно быть TRUE.

OR Оператор:

ИЛИ оператор также используется для объединения нескольких условий WHERE оговорки о SQLite инструкции. При использовании оператора ИЛИ, полное условие будет справедливо, когда предполагается, по крайней мере какое-либо из условий является истинным. Так, например [условие1] ИЛИ [условие2] будет справедливо, если либо условие1 или условие2 верно.

Синтаксис:

Базовый синтаксис оператор ИЛИ с ИНЕКЕМ выглядит следующим образом: 

SELECT column1, column2, columnN 
FROM table_name
WHERE [condition1] OR [condition2]...OR [conditionN]

Вы можете объединить N число условий с помощью оператора OR. Для действий, которые необходимо принять SQLite заявление, будь то транзакции или запрос, только любое одно из условий, разделенных OR должно быть истинным.

SQLite - UPDATE запросов

SQLite UPDATE Query используется для изменения существующих записей в таблице. Вы можете использовать ИНЕКЕ с UPDATE запрос, чтобы обновить выбранные строки в противном случае все строки будут обновлены.

Синтаксис:

Базовый синтаксис UPDATE запроса с ИНЕКЕМ выглядит следующим образом: 

UPDATE table_name
SET column1 = value1, column2 = value2...., columnN = valueN
WHERE [condition];

Вы можете объединить N число условий с помощью И или ИЛИ операторы.

SQLite - запрос на удаление

SQLite DELETE запроса используется для удаления существующих записей из таблицы. Вы можете использовать ИНЕКЕ с DELETE запроса, чтобы удалить выбранные строки, в противном случае все записи будут удалены.

Синтаксис:

Базовый синтаксис DELETE запроса с ИНЕКЕМ выглядит следующим образом: 

DELETE FROM table_name
WHERE [condition];

Вы можете объединить N число условий с помощью И или ИЛИ операторы.
