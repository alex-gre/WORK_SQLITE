SQLite подзапросы

❮ Предыдущая Следующая Глава ❯

Подзапросы или внутренний запрос или вложенный запрос является запросом внутри другого SQLite запроса и заливал в предложении WHERE.

Подзапрос используется для получения данных, которые будут использоваться в основном запросе в качестве условия для дальнейшего ограничения данных, которые будут извлечены.

Подзапросов можно использовать с SELECT, INSERT, UPDATE и DELETE заявления вместе с операторами, как =, <,>,> =, <=, В, МЕЖДУ т.д.

Есть несколько правил, которые подзапросы должны следовать:

  • Подзапросов должны быть заключены в круглые скобки.

  • Подзапрос может иметь только один столбец в ЗЕЬЕСТЕ, если несколько столбцов находятся в главном запросе для подзапроса, чтобы сравнить свои выбранные столбцы.

  • ORDER BY не может быть использован в подзапросе, хотя основной запрос может использовать ORDER BY. GROUP BY может быть использована для выполнения той же функции, что и ORDER BY в подзапросе.

  • Подзапросов, которые возвращают более одной строки могут быть использованы только с несколькими операторами значений, таких как оператор IN.

  • МЕЖДУ оператор не может быть использован с подзапросов; однако, МЕЖДУ может быть использован в подзапроса.

Подзапросы с оператором SELECT:

Подзапросы наиболее часто используется с ЗЕЬЕСТОМ. Базовый синтаксис выглядит следующим образом: 

SELECT column_name [, column_name ]
FROM   table1 [, table2 ]
WHERE  column_name OPERATOR
      (SELECT column_name [, column_name ]
      FROM table1 [, table2 ]
      [WHERE])

Пример:

Рассмотрим COMPANY таблицу оказывает следующие записи: 

ID          NAME        AGE         ADDRESS     SALARY
----------  ----------  ----------  ----------  ----------
1           Paul        32          California  20000.0
2           Allen       25          Texas       15000.0
3           Teddy       23          Norway      20000.0
4           Mark        25          Rich-Mond   65000.0
5           David       27          Texas       85000.0
6           Kim         22          South-Hall  45000.0
7           James       24          Houston     10000.0

Теперь, давайте проверим следующие подзапроса с ЗЕЬЕСТОМ: 

sqlite> SELECT * 
     FROM COMPANY 
     WHERE ID IN (SELECT ID 
                  FROM COMPANY 
                  WHERE SALARY > 45000) ;

Это произведет следующий результат: 

ID          NAME        AGE         ADDRESS     SALARY
----------  ----------  ----------  ----------  ----------
4           Mark        25          Rich-Mond   65000.0
5           David       27          Texas       85000.0

Подзапросы с Заявлением ВСТАВИТЬ:

Подзапросы также может использоваться с операторами INSERT. Оператор INSERT использует данные, возвращаемые из подзапроса для вставки в другую таблицу. Выбранные данные в подзапросе могут быть изменены с помощью любого из функций символов, даты или номера.

Базовый синтаксис выглядит следующим образом: 

INSERT INTO table_name [ (column1 [, column2 ]) ]
           SELECT [ *|column1 [, column2 ]
           FROM table1 [, table2 ]
           [ WHERE VALUE OPERATOR ]

Пример:

Рассмотрим таблицу COMPANY_BKP с аналогичной структурой в виде таблицы COMPANY и могут быть созданы с использованием же CREATE TABLE с помощью COMPANY_BKP в качестве имени таблицы. Теперь, чтобы скопировать всю таблицу COMPANY в COMPANY_BKP, следующий синтаксис: 

sqlite> INSERT INTO COMPANY_BKP
     SELECT * FROM COMPANY 
     WHERE ID IN (SELECT ID 
                  FROM COMPANY) ;

Подзапросы с Заявлением UPDATE:

Подзапрос может быть использован в сочетании с заявлением UPDATE. Либо один или несколько столбцов в таблице, могут быть обновлены при использовании подзапросов с заявлением UPDATE.

Базовый синтаксис выглядит следующим образом: 

UPDATE table
SET column_name = new_value
[ WHERE OPERATOR [ VALUE ]
   (SELECT COLUMN_NAME
   FROM TABLE_NAME)
   [ WHERE) ]

Пример:

Предполагая, что у нас есть таблица COMPANY_BKP доступной, который является резервной копией таблицы COMPANY.

Следующий пример обновляет ЗАРПЛАТУ на 0,50 раза в таблице COMPANY для всех клиентов, чей возраст больше или равен 27: 

sqlite> UPDATE COMPANY
     SET SALARY = SALARY * 0.50
     WHERE AGE IN (SELECT AGE FROM COMPANY_BKP
                   WHERE AGE >= 27 );

Это будет влиять на две строки и, наконец, таблицу COMPANY будет иметь следующие документы: 

ID          NAME        AGE         ADDRESS     SALARY
----------  ----------  ----------  ----------  ----------
1           Paul        32          California  10000.0
2           Allen       25          Texas       15000.0
3           Teddy       23          Norway      20000.0
4           Mark        25          Rich-Mond   65000.0
5           David       27          Texas       42500.0
6           Kim         22          South-Hall  45000.0
7           James       24          Houston     10000.0

Подзапросов с DELETE заявление:

Подзапрос может быть использован в сочетании с ВЕЯТ как с любыми другими высказываниями, упомянутых выше.

Базовый синтаксис выглядит следующим образом: 

DELETE FROM TABLE_NAME
[ WHERE OPERATOR [ VALUE ]
   (SELECT COLUMN_NAME
   FROM TABLE_NAME)
   [ WHERE) ]

Пример:

Предполагая, что у нас есть таблица COMPANY_BKP доступной, который является резервной копией таблицы COMPANY.

Следующий пример удаляет записи из таблицы COMPANY для всех клиентов, чей возраст больше или равен 27: 

sqlite> DELETE FROM COMPANY
     WHERE AGE IN (SELECT AGE FROM COMPANY_BKP
                   WHERE AGE > 27 );

Это будет влиять на две строки и, наконец, таблицу COMPANY будет иметь следующие документы: 

ID          NAME        AGE         ADDRESS     SALARY
----------  ----------  ----------  ----------  ----------
2           Allen       25          Texas       15000.0
3           Teddy       23          Norway      20000.0
4           Mark        25          Rich-Mond   65000.0
5           David       27          Texas       42500.0
6           Kim         22          South-Hall  45000.0
7           James       24          Houston     10000.0
